# 🚀 Proyecto Frontend Web

¡Bienvenidos al repositorio del frontend! Para mantener el código ordenado, evitar conflictos y no borrar el trabajo de los demás, **todo el equipo** debe seguir las siguientes reglas de trabajo.

---

## 🌳 1. Estrategia de Ramas (Solo dos ramas)
Para no complicarnos la vida, trabajaremos únicamente con dos ramas:

* 🔒 **`master` (Producción):** Es la versión estable e intocable del proyecto. **PROHIBIDO HACER PUSH DIRECTO AQUÍ.** Solo se actualiza a través de Pull Requests.
* 🛠️ **`development` (Trabajo):** Es nuestra rama del día a día. Aquí es donde todos vamos a programar, integrar nuestro código y hacer pruebas.

---

## 💻 2. ¿Cómo subir tu trabajo? (El día a día)
Como todos estaremos trabajando sobre la rama `development`, es súper importante que siempre descargues los cambios de tus compañeros antes de subir los tuyos. 

Sigue esta rutina **siempre**:

1. **Antes de empezar a programar, bájate los cambios más recientes:**
